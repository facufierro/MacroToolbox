# MacroToolbox v3.0.0

## Changes

- Reworked the whole structure so it is easier to understand: a **Scope is now just a folder** that groups profiles, and each **profile targets its own application** and owns everything that runs against it — its hotkeys, scripts, overlay items, and states.
- Replaced activate/deactivate with a simple **Armed** switch per profile. An armed profile's hotkeys and scripts work automatically whenever its target app is the focused window, and **several profiles can be armed at once**, each tied to its own app — no more switching one active scope at a time or "waiting for game".
- A profile can be set to **Any app** (the replacement for the old global scope), so its hotkeys work everywhere.
- There is always a **Global** folder — a permanent container that can't be deleted (recreated empty if it ever goes missing). What you put in it is entirely up to you; its profiles can be created, renamed, and deleted freely.
- **Scripts now live on the profile** (alongside its hotkeys) and **run from their own folder**, so a relative path in a script resolves next to the script file instead of some unpredictable location.
- Moved each profile's application, overlay on/off, and toggle keys into a single **profile Settings** dialog; the sidebar and dashboard now show your folders, and the overlay follows whichever armed profile's app is focused.
- Your existing configuration is **migrated automatically** on first launch: each old scope becomes a folder, its profiles inherit the scope's app and scripts, and the profile that was active (or the old global scope's profile) is armed. A backup of the previous `db.json` is written next to it.
- **Redesigned the interface** around the profile: a resizable left library lists every folder (collapsed by default) and its profiles — each with its target app and an inline Armed switch and a status dot (green when its app is running, yellow when armed but the app is closed). Selecting a profile opens a focused, full-width editor. The window fills at any size, and profiles can be searched.
- **Each profile now has a type** — Hotkeys, Scripts, or Overlay — chosen when you create it, so the editor shows just that one thing instead of a tab bar.
- **Right-click a folder or a profile for its actions.** The browser's default right-click menu is gone; instead, right-clicking a folder offers new-profile/import/arm-all/rename/export/delete, and right-clicking a profile offers its settings, parent, copy, edit states, export, borderless, kill, and delete.
- The main window again **opens maximized** on launch (and stays maximized when restored from the tray).
