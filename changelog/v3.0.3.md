# MacroToolbox v3.0.3

## Changes

- Made script shutdown reliable. Each script (and anything it spawns) is now bound to the app through a Windows Job Object, so it is always terminated when you quit MacroToolbox — including when the app crashes or is force-closed — or when you disarm the profile that owns it. The v3.0.2 attempt could still leave a script running (and its folder locked) if the app didn't exit through its normal quit path.
