# Hotkey Manager v3.0.5

## Changes

- Fixed hotkeys sometimes taking a long time to fire (e.g. RAlt+N being slow to type ñ): the key's action now runs before the overlay is notified, and the overlay notification can no longer stall a keypress.
- Fixed a hold-to-repeat hotkey occasionally getting stuck repeating after the key was released.
- The app now checks for updates periodically while running, not just at launch, so a new release is noticed without restarting.
- Fixed updates failing to install when a leftover AutoHotkey process (e.g. after a crash) kept the app's files locked; its AutoHotkey now always exits with the app.
